Var:

1. Variables declared with var have function-level scope. This means they are only visible within the function in which they are declared.
2. If var is used outside of any function, the variable becomes a global variable and is accessible throughout the entire program.
3. Hoisting: Variables declared with var are hoisted to the top of their scope. This means that the variable declaration is moved to the top of the function or global context during the compilation phase.

Let:

01. Variables declared with let have block-level scope. This means they are only visible within the block (enclosed by curly braces) in which they are defined.
02. let allows you to reassign a value to the variable.

Const:

01. Variables declared with const also have block-level scope.
02. const is used to declare constants; once a value is assigned, it cannot be reassigned.
03. While the variable itself is not mutable, the object or array it points to can still be modified.

Null

01. null is often used as an explicit assignment to indicate the absence of a value or to reset a variable or object property.
02. Sometimes, when you want to represent that an object property intentionally has no value, you may assign null to that property.

Undefined

01. When you declare a variable but don't assign a value to it, the variable is automatically assigned the value undefined.
02. Accessing an object property that does not exist returns undefined.
03. If a function is called with fewer arguments than declared parameters, the remaining parameters are set to undefined.
